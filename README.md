# Бизнес-клуб МГУ — лендинг (шаблон)

Это простой, аккуратный вертикальный лендинг на **HTML + CSS + JS** (без зависимостей).
Дизайн — тёмный, с мягкими синими переливами/градиентами и умеренным «неоновым» свечением.

## Структура проекта

```
msu-business-club-landing/
  index.html
  css/
    styles.css
  js/
    main.js
  assets/
    logo.svg
    hero-bg.svg
    product-*.svg
```

## Где что менять

### 1) Тексты и ссылки
- Все тексты находятся в `index.html`.
- Все ссылки-заглушки помечены `href="#"` — замените на реальные.

### 2) Логотип
Файл: `assets/logo.svg`

Рекомендации:
- **SVG** (лучше всего) или **PNG**.
- Размер: **квадрат**, желательно **256×256** или выше.
- Вёрстка делает логотип круглым автоматически.

### 3) Фон заставки (hero)
Сейчас фон подставляется из `assets/hero-bg.svg` (см. `.hero__bg` в `css/styles.css`).

Рекомендации:
- Формат: **WebP/JPG/PNG/SVG**
- Соотношение сторон: **16:9** или **21:9**
- Ширина: от **1600px** (лучше 2000–2400px для больших экранов)

Чтобы заменить:
1. Положите файл в `assets/`, например `hero.jpg`
2. В `css/styles.css` поменяйте:
   ```css
   .hero__bg{
     background-image: url("../assets/hero.jpg");
   }
   ```

### 4) Картинки продуктов
Сейчас используются SVG-заглушки:

- `assets/product-mentorship.svg`
- `assets/product-intensives.svg`
- `assets/product-networking.svg`
- `assets/product-career.svg`

Рекомендации:
- Соотношение сторон: **4:3**
- Размер: от **1200×900**

Можно заменить в `index.html` в карточках продуктов.

### 5) Форма заявки (всплывающее окно снизу)
Форма находится в `index.html` в блоке `#applyModal`.

- Открывается по кнопкам с классом `.js-open-apply`
- Закрывается по клику на затемнение/кнопку ✕/Escape
- Отправка сейчас **демо**: данные выводятся в консоль браузера

Логика в `js/main.js`, ищите:
- `applyForm.addEventListener("submit", ...)`

## Запуск

Самый простой вариант:
- Открыть `index.html` двойным кликом.

Для локального сервера (рекомендуется):
- VS Code → расширение **Live Server**
- или любой простой static-server.

## Подключение реальной отправки форм
В `js/main.js` замените «демо-отправку» на `fetch()` к вашему API
или на интеграцию с Google Forms/Telegram ботом.

> Важно: если собираете персональные данные, добавьте корректный текст согласия/ссылку на политику обработки.
